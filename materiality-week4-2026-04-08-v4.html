How Thresholds Work

Full explanation
GREEN — OK
Within normal range. System logs the value but no alert fires. No action needed.
Example: Sterling flood = 0 facilities in FEMA hazard zone (threshold: ≤0)
YELLOW — Monitor
Value crossed into watch zone. Email to deal lead only with a monitoring action.
Example: Sterling CBAM = $186k (threshold: >$150k). Action: brief James on CBAM trajectory.
RED — Act Now
Threshold breached. Email to deal lead + Sarah Chen (CC) with specific action + MACC initiative attached.
Example: Trident energy = 77.9% YoY (threshold: ≥10%). Action: cold-aisle retrofit + PPA.
Transition-only alerting. Emails fire only when a portco crosses a band boundary — not on every 15-minute scan. If Sterling is YELLOW today and still YELLOW tomorrow, no new email. 24-hour cooldown per alert. All thresholds are configurable in a single JSON file — no code changes.
How it works. A Python engine fetches live data from 6 APIs (FEMA, WRI Aqueduct, EIA, ECB, Open-Meteo heat + precipitation), evaluates each portco against its threshold config, and publishes the state. A launchd schedule runs it every 15 minutes. Fresh transitions fire email alerts via Resend (or write .eml files in dry-run mode).
